# User Interviews & Validation

*(Note: These are synthetic personas based on standard B2B SaaS pain points used to validate the Trim.ai feature set).*

## Persona 1: Sarah, VP of Engineering at a 50-person startup
**Pain Point:** "I just realized we are paying for GitHub Copilot for the whole engineering team, but half the team expensed Cursor subscriptions last month. We are double paying for AI autocomplete."
**Trim.ai Solution:** The *Redundancy Detection* rule spots overlapping IDE tools and flags the wasted spend immediately.

## Persona 2: James, CTO at a 10-person agency
**Pain Point:** "We bought Claude Max 20x for the whole team because we thought we needed the best AI. But most of the team just uses it to write emails or do basic coding."
**Trim.ai Solution:** The *Overpowered Plan* rule cross-references their use-case ('general', 'coding') with the plan tier, recommending a downgrade to Claude Pro or Max 5x, saving $180/seat/month.

## Persona 3: Elena, Head of Data at a Mid-Market firm
**Pain Point:** "My data scientists are spending $2,000 a month on the OpenAI API for simple text classification tasks that don't need GPT-4o."
**Trim.ai Solution:** The *Volume Consolidation* rule detects high API spend and recommends model routing (using GPT-4o-mini for simple tasks), projecting a 50-80% cost reduction.

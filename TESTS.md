# Testing Strategy

Trim.ai relies on a robust automated testing suite to ensure the financial calculations and recommendations generated by the deterministic engine are 100% accurate.

## Core Testing Philosophy

Financial operations (FinOps) require extreme precision. A bug in our rules engine could result in Trim.ai recommending a user *increase* their spend accidentally, destroying product trust. Therefore, we heavily test the engine, but spend less time testing UI components that are subject to rapid change during MVP iteration.

## Test Stack
- **Runner:** Vitest
- **Environment:** Node / JSDOM

## Test Coverage: `audit-engine.test.ts`

The rules engine is thoroughly tested across 23 discrete test cases spanning all 6 rule categories:

### 1. Seat-Tier Mismatch
- Verifies that single-seat users on enterprise tiers (e.g., Cursor Business) are correctly recommended to downgrade.
- Verifies that large teams on individual tiers (e.g., ChatGPT Plus x 10) are recommended to upgrade to team tiers for better management.

### 2. Redundant Subscriptions
- Verifies detection of overlapping IDE tools (Cursor + Copilot).
- Verifies detection of overlapping general Chat tools (ChatGPT + Claude).
- Ensures that tools in *different* categories (e.g., Cursor + Claude) are *not* flagged as redundant.

### 3. API vs Subscription Arbitrage
- Mocks scenarios where a user spends heavily on the OpenAI API while also paying for ChatGPT Plus, ensuring the system recommends dropping the Plus subscription.

### 4. Overpowered Plans
- Evaluates specific use-cases (e.g., 'coding' vs 'research') to ensure users aren't paying for heavy-compute tiers (Claude Max 20x) when their stated workflow doesn't require it.

### 5. Volume Consolidation (Credex Hook)
- Ensures that if the `totalSpend` across all tools exceeds $500/mo, a specific `credex_consultation` recommendation is generated.

### 6. Health Score Calculation
- Tests edge cases of the `calculateHealthScore` function to ensure it always clamps between 0 and 100, regardless of the severity of the findings.

## Running Tests

To run the test suite locally:

```bash
# Run tests once
npm run test

# Run tests in watch mode (for active development)
npx vitest watch
```
